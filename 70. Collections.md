
## Коллекции

```bash
# Проверьте ping
ping galaxy.ansible.com
```

```bash
# Проверьте доступность через curl
curl -I https://galaxy.ansible.com

# Установка
ansible-galaxy collection install ansible.posix --upgrade --force
ansible-galaxy collection install ansible.posix --upgrade --force --timeout 60

# Или используйте git
ansible-galaxy collection install git+https://github.com/ansible-collections/ansible.posix

# Скачайте напрямую
wget https://galaxy.ansible.com/api/v3/plugin/ansible/content/published/collections/artifacts/ansible-posix-2.1.0.tar.gz

# Установите локально
ansible-galaxy collection install ./ansible-posix-2.1.0.tar.gz
```


## Проверить установленные коллекции
```bash
# Показать все установленные коллекции
ansible-galaxy collection list

# Показать конкретную коллекцию
ansible-galaxy collection list ansible.posix
```


## Проверить доступные модули из коллекции
```bash
# Показать все модули из коллекции
ansible-doc -t module -l | grep posix

# Или посмотреть конкретный модуль (например, firewalld)
ansible-doc ansible.posix.firewalld
```


## Тест конкретных модулей
```bash
# Проверить модуль synchronize (аналог rsync)
ansible localhost -m ansible.posix.synchronize -a "mode=pull src=/etc/hosts dest=/tmp/hosts.copy" --connection=local -v

# Проверить модуль seboolean
ansible localhost -m ansible.posix.seboolean -a "name=httpd_can_network_connect state=yes persistent=yes" --connection=local
```

## Полный список модулей коллекции
```bash
# Посмотреть все доступные модули
find /root/.ansible/collections/ansible_collections/ansible/posix/plugins/modules/ -name "*.py" | wc -l

# Вывести имена модулей
find /root/.ansible/collections/ansible_collections/ansible/posix/plugins/modules/ -name "*.py" | xargs -I {} basename {} .py
```
